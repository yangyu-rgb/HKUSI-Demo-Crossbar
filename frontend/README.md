# CrossBorder AI 前端

用于 CrossBorder AI 本地 Demo 的 React、TypeScript 和 Vite 界面。

## 架构

```text
src/
  layout/     # 共享导航、重置控制和页面外壳
  pages/      # 懒加载的路由级页面
  features/   # 功能 API、查询 Hook、组件和局部类型
  generated/  # 从后端 OpenAPI 契约生成的类型
  shared/     # API 错误、查询键、格式化、Skeleton 和页面状态
```

TanStack Query 负责服务端状态、缓存失效、Mutation 和 60 秒实时轮询。全局时钟基于服务端香港时间每秒推进，规划和企业表单使用服务端建议时间初始化。

路由：

- `/`：游客可直接查看的四口岸状态、手动刷新和 Recharts 趋势对比。
- `/planner`：登录后可用的固定地点搜索、显式预测、引导空状态、可解释计算弹窗和 SVG 路线。
- `/crowdsource`：课堂反馈、方向/通道、重复提交拦截、质量评分、多人共识与15%/30%/45%动态校准。
- `/alerts`：持久化订阅 CRUD、下一次通勤评估、本地告警周期、通知收件箱和已读状态。
- `/business`：企业预测与调度方案；支持巴士、物流、企业客户和口岸官方视角，展示实际 HGB 推理的模型版本/覆盖率/90% 区间、透明降级、任务/车辆调整、采用前后对比、本地通知草稿、执行清单和事后复盘。
- `/business/employees`：保留原员工 CSV 导入校验、偏好编辑、风险/口岸筛选、方案历史和 CSV 导出，作为企业客户辅助流程。
- `/model`：V2.2 技术公式、大白话解释、基础模型指标、区间校准和 V1 影子对照。
- `/operations`：仅 Demo 操作员可访问的预测、众包、错误、审计和适配器运营分析。
- `/login`：本地身份选择、角色工作空间和安全站内回跳，不执行真实 OAuth 或密码认证。
- `/pricing`：游客可比较三档套餐；登录后执行月付/年付模拟结账、本地收据和商业订阅状态。
- `/mobile/login`：独立移动登录页，固定进入个人通勤空间。
- `/mobile`：登录后的独立手机首页，提供当前最优口岸、四口岸排名、双侧来源说明和四个课堂快捷入口。

首页提供懒加载的可交互 Three.js 港深四口岸微缩城市沙盘：本地地理资产包含 OpenStreetMap 的港深区域轮廓、海岸/行政边界与经过筛选的 motorway、trunk、primary 道路，并保留 ODbL 元数据和可见署名。统一经纬度投影生成挤压地形、海面、道路骨架、四类程序化建筑、抽象地标剪影和带大厅/车道/雨棚/闸机的口岸节点。四条三层曲线路线使用颜色表达压力、粒子密度表达人流、速度表达效率；双向粒子分离并以错峰批次流动，非线性进度映射使高压路线在口岸两侧更明显地减速累积，全部状态变化保持平滑。场景支持受限旋转/平移/缩放、模块内跟随详情、平滑聚焦、返回总览和流畅/均衡/精细画质；任意相机操作会暂停巡航8秒，方向键提供键盘平移，减少动态效果时关闭连续巡航和镜头飞行。固定 Raycast 目标在初始化时缓存；场景离开视口、页面隐藏或 WebGL context lost 时取消 RAF，恢复后只启动单一循环并保留路线选择。地理资产失败时降级为旧轮廓，无 WebGL 时降级为完整文字数据，不重复提供 2D 地图。移动端提供返回网页版、PWA manifest、生产构建安装入口、在线状态和离线壳；离线时不会缓存或伪造预测 API 响应。动态画质与正式 Web Vitals 不属于课堂 Demo 范围，企业实时全屏大屏留待真实产品目标明确后评估。

桌面端使用 Inter 和 CSS Modules 实现黑灰白电影感设计系统。首页 Hero 播放指定远程城市视频原画，不叠加全屏暗层，并以 `/hero-city-poster.jpg` 处理断网、视频失败和减少动态效果场景；背景运动超过五秒，因此提供显式暂停/播放控制。标题、内容区和路由转场使用 `src/shared/motion/` 的共享组件，统一尊重 `prefers-reduced-motion`。其余桌面页面只使用深色表面和液态玻璃，不加载背景视频或装饰图片；`/mobile/*` 保持原有独立手机视觉，只继承全局 Inter 字体。

桌面导航按公开浏览、个人、企业管理员、运输调度、口岸官方和平台运营角色显示对应入口，其余当前身份可见功能集中到“更多”；未登录受保护入口继续安全跳转登录。菜单支持 Escape、点击外部关闭、键盘焦点和 `aria-expanded`，页面提供跳到主要内容链接。企业方向切换只复用既有页面外壳、卡片、表格和按钮样式，没有建立新的视觉布局。所有视觉修改仍受 `docs/ui_redesign_guardrails.md` 的信息合同约束。

网页版只有口岸态势和套餐公开。登录后个人身份可使用规划、众包、提醒和 AI；企业管理员可使用运营控制塔、员工接驳、规划和 AI；巴士/物流调度身份进入各自运营工作空间；口岸官方只查看聚合压力和协调建议；平台运营可切换四类企业视角并使用全部桌面功能。无权访问时显示权限说明，不泄露目标页面。身份切换通过退出后重新登录完成。有效本地会话的请求会携带 `X-Demo-Persona-ID`；这不是生产登录系统。

## 运行

从仓库根目录执行以下命令，可以同时启动前后端：

```bash
./start.sh
```

需要单独调试前端时，先在 `8000` 端口启动后端，然后在 `frontend/` 目录执行：

```bash
npm install
npm run dev
```

需要修改 API 地址时：

```bash
VITE_API_BASE=http://127.0.0.1:8000 npm run dev
```

3D 地理资产已提交在 `public/data/hk-shenzhen-geography.json`，正常启动不依赖外网。只有明确刷新来源快照时才运行以下维护命令；刷新后必须检查来源许可、视觉边界和差异，固定 `SOURCE_DATE_EPOCH` 可保持输出时间戳可复现：

```bash
SOURCE_DATE_EPOCH=1783900800 npm run generate:geography -- --refresh
```

资产来源为 OpenStreetMap Nominatim 与 Overpass API，遵循 ODbL 1.0，并在页面和资产元数据中保留 `© OpenStreetMap contributors`。该资产经过裁剪、简化和限量，仅作课堂态势可视化，不是测绘或导航数据。

## 验证和重新生成契约

```bash
npm test
npm run test:coverage
npm run build
npm run generate:api
npm run test:e2e
```

`test:coverage` 对关键 API、众包、认证和运营模块执行不低于70%的语句、分支、函数和行覆盖率门槛。`npm run check:api` 会重新生成 OpenAPI 文件；如果已提交的契约文件不是最新版本，命令会失败。Playwright 会分别启动并等待后端存活探针与前端页面，再用 Chromium 覆盖视频失败封面、电影感外壳断点、交互式四口岸流线、首次无方案的双向规划、计算弹窗、场景实验室、通知、运营分析、模型说明和独立手机首页，并对全部业务路由执行 axe WCAG A/AA 检查。Playwright 固定使用单 worker，避免并行重置共享 Demo SQLite 状态。后续视觉维护必须遵守 `docs/ui_redesign_guardrails.md` 的信息与功能保护边界。

Topic 2 的主演示回归还覆盖企业 AI 决策闭环：HGB 全覆盖客运场景、货运部分覆盖与显式降级、3→0 风险对比、一键采用、147 条本地通知草稿以及口岸官方协调建议。对应英文讲稿见 `../docs/demo_script_7min_en.md`。
