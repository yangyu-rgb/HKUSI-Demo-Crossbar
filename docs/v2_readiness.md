# 真实数据研究与生产晋级前置检查

本文件描述真实数据研究和生产晋级门槛，不再阻止课堂 Demo 使用可解释合成数据训练的 AI v2。课堂 AI v2 明确标注为 `synthetic_scenario_classroom_demo`，用于展示“场景变化—等待预测—路线推荐”能力，不代表真实口岸准确率。

当前系统还没有训练或启用 V2。本阶段的目标是把每一条可用实际等待反馈追溯到产生它的统计预测和影子 AI 值，并防止合成数据或低质量反馈被误当作模型效果证据。

## 标签闭环

1. `POST /api/predict` 生成确定性的 `forecast_run_id`，并保存每个口岸的统计预测、影子输出、特征、数据版本和来源状态。
2. 路线页面将该运行 ID 与推荐口岸预填到众包页面。
3. `POST /api/crowdsource/report` 在验证口岸匹配后保存反馈，并记录通关方向、通关类型、等待起止时间、来源、是否为实际现场观察以及建模同意。
4. 只有声明为实际现场观察、获得建模同意、来源为真实众包/合作方/官方、未过期、`high` 质量且该运行口岸尚无标签的记录会写入实际等待标签。
5. `python backend/scripts/export_training_snapshot.py` 在 `data/runtime/training_snapshots/` 生成 CSV 和 JSON 元数据。输出目录被 Git 忽略。

快照 schema v3 包含标签哈希、等待起止时间、方向/通道、来源、建模同意、真实日期时间切分、Provider 状态、每条标签的 `forecast_run_id` / 报告 ID，以及预测生成时已经冻结的官方居民/访客等级与每日客流。老预测运行没有冻结字段时明确导出为缺失，不用当前值回填。演示种子、普通 Demo 录入、低质量或未授权反馈只保留审计关联，不进入训练 CSV。它可复核数据来源，但不能自动证明模型有效。

真实现场反馈的质量评分使用新鲜度、等待起止元数据和人流一致性，不会因为“更接近模拟基线”而获得更高训练资格，避免用现有模型假设筛掉真实偏差。

## 受控实验阈值

`GET /api/demo/v2-readiness` 和企业页使用相同检查：

- 至少 200 条来源可追溯且获得建模同意的高质量实际等待标签；
- 覆盖四个口岸；
- 至少 21 个独立日期和 8 个小时切片；
- 口岸、天气、日历和事件本地 Provider 均未降级；
- 训练、验证和测试必须按日期先后划分，禁止随机打散未来标签。
- 三个时间分区必须都有样本。

200 条只是启动受控实验的最低门槛，不代表四口岸 × 21 日期 × 8 小时的 672 个组合单元已完整覆盖。就绪接口会额外报告已关联但被隔离的数据、真实训练来源和口岸分布集中风险。

## 生成数据政策

- 现有 5,376 条生成历史样本继续用于 V1 工程流水线、回归测试、压力情景和消融对比。
- 首个 V2 候选以真实标签为主要训练数据；生成数据不得进入真实验证集、测试集或晋级指标。
- 只有未来采用确有迁移能力的模型架构时，才可把生成数据作为可移除的初始化实验；结果必须与“仅真实数据训练”版本独立比较。
- 当前统计/梯度提升表格模型不把大量生成数据训练包装为“预训练”。

所有检查通过时，只表示可以开展本地受控实验，仍不表示可以替换统计预测器。生产晋级还必须具备经审核的真实数据源、独立回测、按口岸和时间切片的误差/校准比较、漂移监测和人工验收。

候选外部来源必须先完成 [`real_data_intake.md`](real_data_intake.md) 的授权、隐私、语义和质量审查；当前公共众包 API 不能把提交者声明为官方或合作方来源。

## 官方特征区

香港入境处陆路口岸等候状态和每日客流已通过独立采集器进入官方特征区。等候状态只包含 `normal`、`busy`、`very_busy`、维护和非服务时间等等级；每日客流是日期级数量。两者都不具备精确等待起止时间，因此：

- 不写入 `actual_wait_minutes`，不增加 200 条标签计数；
- 居民和访客状态保持独立，不用区间中点伪造分钟；
- 原始响应、抓取时间、来源版本、内容哈希和字段映射可审计；
- 来源修订以追加记录保存；预测只使用生成时刻已经获知的值，并把结果冻结到运行记录；
- `GET /api/demo/v2-readiness.external_data` 单独报告特征观测、口岸/方向/日期/小时覆盖、24 小时采集完整率/最大缺口、预测快照覆盖和官方序数等级一致性。

从仓库根目录运行 `./collector.sh start` 启动独立本地采集 sidecar，使用 `./collector.sh status` 查看进程与 24 小时指标，使用 `./collector.sh stop` 停止。也可运行 `backend/.venv/bin/python backend/scripts/collect_official_sources.py` 单次采集。采集不随 Demo 启动自动执行，网络失败也不会改变 V1 输出。

官方一致性把 V1 等待值映射到居民/访客各自的公开等级阈值，只计算类别命中率、平均序数误差和混淆矩阵。它不把等级中点伪造成真实分钟，不计算分钟 MAE，也不参与 200 条门槛。

候选精确分钟 CSV 先运行 `python backend/scripts/validate_exact_wait_labels.py FILE.csv`。只有登记为 `approved_label` 且具备批准批次、稳定记录 ID、带时区等待起止时间、口岸和方向的记录才可能通过；该命令只读校验，不导入数据库。
