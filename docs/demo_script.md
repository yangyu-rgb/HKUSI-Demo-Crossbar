# CrossBorder AI Demo 演示脚本

目标时长：4 分钟。

## 开场——20 秒

“普通地图能规划交通，却看不到未来口岸排队的不确定性。CrossBorder AI 将口岸等待预测、众包校准和主动提醒放进同一个跨境决策流程。”

明确说明：页面时间为香港实时时间；口岸、天气和交通使用本地模拟数据，不代表真实口岸情况。

## 场景一：口岸态势——35 秒

1. 打开 `/`。
2. 比较当前等待卡片和四条折线组成的未来三小时趋势图。
3. 指出全局香港时钟、60 秒数据刷新、手动刷新以及模拟天气和交通提示。

核心信息：产品不仅显示现在是否拥堵，也展示拥堵趋势。

## 场景二：可解释路线决策——85 秒

1. 打开 `/planner`。
2. 选择香港大学 → 深圳南山科技园，保留动态建议到达时间，预算 HK$100。
3. 使用“稳妥均衡”生成建议。
4. 展示推荐 SVG 路线、最晚出发、预计到达、缓冲、动态 90% 区间、风险和费用。
5. 展开“查看预测依据”，展示时间匹配历史基线、样本数量和衰减后的众包权重。
6. 修改出发地、目的地、偏好或预算并重新生成。

核心信息：系统回答“何时出发”和“为什么这条路线更稳”，依据来自可审查的 Demo 统计计算，而不是黑箱结论。

## 场景三：持久化众包闭环——40 秒

1. 打开 `/crowdsource`。
2. 输入实际等待时间和可选说明；表单不预填虚假输入。
3. 提交并展示动态积分、质量分、可信等级和有效截止时间。
4. 再次向同一口岸提交，展示 10 分钟重复提交拦截。
5. 返回 `/planner` 重新生成，展示有效众包数量和预测依据中的平均质量分。
6. 说明过期反馈仍保存在 SQLite，但不会继续显示或参与预测，直到执行 Demo 重置。

核心信息：用户现场反馈闭合预测校准循环。

## 场景四：订阅 CRUD——35 秒

1. 打开 `/alerts`。
2. 创建提醒并在已保存列表中查看。
3. 修改路线或时间，也可以删除订阅。
4. 说明出发提醒、异常拥堵和更优路线三类设置。

核心信息：主动服务包含可管理、可持久化的订阅，而不只是一次成功提示。

## 场景五：可编辑企业方案——40 秒

1. 打开 `/business`。
2. 编辑员工、增加一名员工并生成方案。
3. 展示汇总风险和员工级出发建议。
4. 再次生成，然后从近期方案历史中载入一次输入。

核心信息：同一预测能力可以支撑可重复的企业调度流程。

## 结束与重置——25 秒

“CrossBorder AI 的商业重点不是向个人用户收导航费，而是把跨境等待风险转化为巴士、物流和企业 HR 可执行的调度服务。”

下一次演示前使用页头重置按钮，按当时香港时间恢复反馈和订阅种子并清空企业历史。

## 备用材料

- 60–90 秒屏幕录制。
- 每个路由页面一张截图。
- 一张架构图：缓存 JSON + SQLite → Service → API → 查询 Hook。
- 本地服务失败时使用 `docs/openapi.json` 和静态种子数据。
