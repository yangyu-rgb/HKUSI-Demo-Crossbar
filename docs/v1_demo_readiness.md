# V1 Demo 完成标准

V1 的目标是形成可重复、可解释、可重置的本地演示闭环，而不是模拟生产环境。`GET /api/demo/v1-readiness` 和页面 `/model` 是统一检查入口；`GET /api/health/live` 与 `GET /api/health/ready` 用于进程和依赖探测。

## 已闭合的业务流程

1. 首页读取四口岸状态、趋势和本地输入来源。
2. 规划页支持香港→深圳与深圳→香港，比较四个口岸并解释时间、费用、风险、置信区间与因素权重。
3. 众包页保存课堂反馈、质量和来源信息并校准当前预测；不会把 Demo 输入包装成真实训练标签。
4. 提醒页完成订阅 CRUD、下一次通勤评估、本地告警周期、通知收件箱和已读状态。
5. 企业运营页完成巴士/物流/口岸角色视角、风险预览、方案采用、本地通知草稿、执行清单和人工复盘；员工 CSV 接驳流程保留为辅助入口。
6. 模型页展示 AI v1 合成数据评估、分口岸指标、影子差异、V1 就绪度和独立的 V2 数据门槛。

## Demo 控制与验证

- 六种本地身份：平台运营、跨境通勤者、企业管理员、巴士调度、物流调度和口岸官方。企业方案按 `organization_id` 隔离，官方视角不返回企业任务明细，企业功能和审计接口执行角色检查。
- SQLite 保存反馈、预测运行、订阅、评估、通知、企业运营方案、协调建议、员工接驳方案、影子观测与审计；页头可一键重置动态状态。
- `start.sh` 在启动前确保被 Git 忽略的 AI v1 运行时产物存在且与已跟踪元数据和训练数据哈希兼容。
- 所有写操作产生本地审计事件；响应包含请求 ID 和基础安全响应头。
- 后端 Pytest、前端 Vitest、TypeScript/Vite 构建、OpenAPI 漂移检查、桌面/移动端 Playwright 与 axe WCAG A/AA 检查均进入 CI。

## 不属于 V1 的生产能力

- 真实口岸、交通、天气、地图或事件数据接入。
- 生产认证、邮件/短信/推送、数据库、任务队列、缓存、部署和监控。
- 用 AI v1 替换用户可见的统计预测，或用合成数据宣称真实效果。
- 现场真实标签收集与生产晋级已退出课程范围；众包只用于课堂预测校准。
