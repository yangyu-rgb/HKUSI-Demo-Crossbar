# CrossBorder AI
## 深港跨境通勤与口岸等待智能规划平台

SIUS2612 Topic 2 AI Business Prototype

---

## 项目简介

CrossBorder AI 是一个专为深港跨境场景设计的 AI 出行与调度平台。它不是再做一个普通地图 App，而是解决一个普通地图很难彻底解决的问题：**跨境路线不只取决于地理和交通，还取决于"口岸排队不确定性"和"车费""，近还不代表低成本的跨境路线。**

深港跨境客流规模大、波动强、场景高度复杂化。上班族需要知道哪条路线更稳，物流公司和跨境巴士公司需要批次安排和风险管理，但目前不深港路线预测并不准确且系统级风险管理决策不能从公开交通导航系统中获取。

本项目的创新点是把跨境"静态路线规划"升级为**"预测驱动的跨境路线规划"**：利用AI预测未来1-3小时口岸等待时间、并给出合路线建议、主动推送最佳出发时间，而非主推C端收费，而是主推B2B/B2G：跨境巴士、物流、企业 HR、园区/企业通勤数据服务。

> **项目定位：** 本仓库只用于 SIUS2612 课堂 Demo。目标是稳定展示“数据—模型—透明校准—路线决策”，不投入生产、不收集现场真实训练标签，也不声称预测值是真实通关分钟。

## 核心功能

### 基础功能层
✅ **时间驱动信息聚合**
- 使用香港实时时钟，动态计算4个口岸（罗湖、福田、皇岗、深圳湾）的模拟等待趋势
- 显示口岸开放状态、特殊通道、模拟天气和交通提示

### 核心差异化层（AI驱动）

🎯 **AI预测引擎**
- 未来1-3小时口岸等待时间预测
- 置信区间和风险量化（"虽然最快但波动大，迟到风险20%"）
- 多模式路线优化（港铁+罗湖 vs 高铁+福田 vs 深圳湾）

✨ **众包智能反馈系统**（创新点）
- 用户实时报告："我现在在罗湖，实际等了15分钟"
- 系统根据反馈质量和新鲜度动态校准预测结果
- 显示："23分钟前有用户报告：福田排队人少"
- 激励机制：报告数据获得积分/VIP功能

✨ **智能提醒订阅**（创新点）
- AI学习用户通勤模式（每周三上午去深圳、偏好最快路线）
- 主动推送："根据当前预测，建议你在8:15出发走福田口岸"
- 异常主动提醒："你常走的罗湖今天异常拥堵+40%"
- 可设置"到达deadline"，系统倒推最晚出发时间

### 商业化功能层（B2B）

💼 **企业批量调度Dashboard**
- 物流公司、跨境巴士公司批量规划
- 员工通勤风险管理和成本优化
- CSV 导入校验、风险/口岸筛选和方案 CSV 导出

### V1 Demo 完整性

- 香港→深圳与深圳→香港双向路线均可规划，地点和四口岸交通矩阵启动时校验。
- 三种本地 Demo 身份覆盖运营、个人通勤与企业管理员；企业数据按组织隔离，敏感操作有本地审计记录。
- 提醒评估可进入本地通知收件箱并标记已读，不伪装成真实邮件、短信或推送。
- `/model` 同时提供技术公式和非技术版解释；`/mobile/*` 是独立手机个人应用，可完成规划、场景推演、众包反馈、提醒通知和模型简报，不跳回桌面业务页。

## AI v2.2 怎样计算

### 技术版

1. 基础模型使用 HistGradientBoosting：

   `B = HGB(口岸、方向、小时、星期、周末、香港官方客流压力)`

2. 天气、节假日和事件使用透明系数，不藏在树模型中：

   `S = B × min(2.10, F天气 × F节假日 × F事件)`

   - 天气：晴天 1.00、下雨 1.08、暴雨 1.18、雷暴 1.25。
   - 节假日：普通日 1.00、节假日 1.24。
   - 事件：无 1.00、低 1.08、中 1.20、高 1.38。

3. 新鲜香港官方拥堵等级继续校准：

   `Q = S × [1 + W官方 × (F等级 − 1)]`

   畅通、繁忙、非常繁忙系数为 0.92、1.08、1.25；权重随数据年龄和预测距离衰减。

4. Demo 众包最多以 30% 权重参与：

   `P = Q × (1 − W众包) + 最新有效众包值 × W众包`

5. 深圳市口岸办公开快照不与香港客流相加。系统比较两侧标准化压力，一致度较低时最多把不确定性扩大 35%，并在结果中提示；这样可以交叉核验来源，又不会把同一批旅客计算两次。

### 大白话版

AI 先看“过去同一个口岸、同一个方向、差不多这个时间和客流量时，通常要等多久”，得到一个底数。然后暴雨、节假日或突发活动按公开系数调整；最新官方拥堵状态和同学输入的课堂众包再继续修正。香港数据负责主要计算，深圳官方数据负责“对答案”；两边差得越大，系统越保守地放宽预测范围。最后把到口岸、排队和过关后的交通时间加起来，比较四条路线。

每个路线卡都显示 AI 基础值、场景值、官方校准值、众包变化、深圳核验一致度和最终结果，属于流程级可解释模型。

## 项目结构

```text
backend/
  app/
    api/                  # FastAPI 路由与依赖
    ml/                   # V1 影子模型、V2.2 模型加载与深圳核验
    repositories/         # JSON/CSV 输入与 SQLite 持久化
    schemas/              # Pydantic API 契约
    services/             # 预测、透明校准和业务流程
    clock.py              # 香港时钟抽象
    main.py               # 应用组装
data/
  history/                # 香港官方客流快照、深圳核验快照与 V1 样本
  models/                 # 离线模型元数据（不包含运行时二进制）
  realtime/               # 口岸模拟元数据
  routes/                 # 地点与交通矩阵
  factors/                # 模拟天气、节假日和事件
  crowdsource/            # 相对当前时间的反馈种子
  sources/                # 官方来源登记与用途边界
  runtime/                # 被忽略的 SQLite、训练集和模型二进制
docs/                     # API 契约和演示说明
frontend/
  src/
    features/             # 功能 API、查询 Hook 和组件
    generated/            # OpenAPI 生成类型
    layout/               # 全局导航和香港时钟
    pages/                # 七个桌面业务页面和移动首页
    mobile/               # 独立移动布局、会话状态和个人业务页面
    shared/               # API、格式化和共享状态组件
```

## 技术栈

### 前端
- **框架**: React 19 + TypeScript
- **构建工具**: Vite
- **服务端状态**: TanStack Query
- **图表**: Recharts（预测折线图、置信区间可视化）
- **实时通信**: 60 秒轮询

### 后端
- **框架**: FastAPI (Python 3.10+)
- **数据验证**: Pydantic
- **数据**: JSON + CSV + 本地 SQLite

### AI/数据
- **课堂主模型**: AI v2.2 基础 HGB + 透明场景/官方/众包校准
- **离线 AI v1**: HistGradientBoosting 等待时间回归 + 时间切分评估
- **真实特征**: 香港入境处每日四口岸双向客流；深圳市口岸办公开统计只作独立核验

AI v1 冻结为影子对照。AI v2.2 使用最近 730 个完整日期、140,160 条确定性场景样本训练基础等待模型；等待分钟目标仍由可解释公式生成。模型或哈希异常时自动回退统计模型。离线指标只证明课堂流水线稳定，不代表真实口岸准确率。

项目不执行现场真实标签收集。众包表单只用于课堂演示的即时校准，启动也不依赖外部采集器；所有关键输入均可从仓库快照复现。

## 快速开始

在仓库根目录执行：

```bash
./start.sh
```

脚本会在缺少依赖时自动创建后端虚拟环境并安装前后端依赖，然后同时启动：

- 前端：`http://127.0.0.1:5173`
- API 文档：`http://127.0.0.1:8000/docs`

按 `Ctrl+C` 会同时停止前后端服务。需要单独调试某个服务时，参见 `backend/README.md` 和 `frontend/README.md`。

该脚本面向 macOS 和 Linux；Windows 环境可分别使用前后端说明中的命令。

## 演示场景

### 场景1：个人用户 - 跨境学生
**小明：香港大学学生，需要9:30前到达深圳南山实习**

1. 打开首页，看到4个口岸实时状态
2. 看到众包反馈："5分钟前 @user123: 福田人很少"
3. 输入查询：9:30到达南山
4. AI推荐：福田口岸（预测等待15分钟，置信区间12-20，风险低）
5. 通关后提交反馈："实际等了12分钟" → 获得10积分
6. 设置智能提醒：每周一三五自动推送最优路线

### 场景2：智能提醒触发
**周三早上7:45**
- 小明收到推送："建议8:15出发走福田口岸"
- 显示预测图表和完整路线

**周五早上检测到异常**
- 7:30推送："⚠️ 罗湖今天异常拥堵+40%，建议改走深圳湾"

### 场景3：企业批量调度
**跨境物流公司HR：20名员工需要9点前到达深圳**

1. 上传员工列表和通勤需求
2. 系统批量预测并生成优化调度方案
3. Dashboard显示：平均通勤75分钟，3人存在迟到风险>20%
4. 系统建议：3名风险员工提前30分钟出发

```bash

frontend/<feature-name>   # 前端功能分支
backend/<feature-name>    # 后端功能分支
```
