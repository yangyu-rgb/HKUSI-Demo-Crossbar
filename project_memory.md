## 当前状态

- 最终产品方向：**CrossBorder AI**，用于深港跨境运输预测、企业调度和口岸协同的 Demo；个人规划作为保留能力，不是商业主演示。
- 本地分支：`main`，跟踪 `origin/main`。
- 2026-07-09 已成功刷新远端引用；模块化工作开始前，本地与远端均位于 `d29c228`。
- 后端：采用 Router、Schema、Service、Repository 分层的 FastAPI 模块化单体，具备最小 Provider/Repository 协议、可操作的类型化错误、本地错误事件和 SQLite 持久化。
- 前端：网页版口岸态势与套餐允许游客查看，其余桌面功能通过本地会话和统一角色守卫开放；独立 `/mobile/login` 只允许个人通勤身份进入 `/mobile/*`，手机端提供返回网页版、退出、安装式 PWA 壳、在线状态、规划、场景、反馈、提醒通知和模型简报。
- 路线规划：香港→深圳与深圳→香港均可用；可解释时间加权统计模型按工作日/周末、小时、模拟天气和 28 天历史半衰期生成基线，并以衰减众包修正、动态不确定性计算 90% 置信区间和迟到概率。
- 数据：四口岸模拟元数据、交通矩阵、5,376 条生成历史样本、模拟天气、相对时间众包种子、确定性巴士/货运运营情景，以及被 Git 忽略的 `data/runtime/` SQLite 数据库。
- 用户流程：企业控制塔可完成风险发现、决策预览、方案采用、本地通知草稿、执行清单与人工复盘；个人端继续提供双向预测、众包、提醒和通知闭环，原员工 CSV 流程保留在辅助路由。
- 文档：API 契约、OpenAPI Schema、生成的 TypeScript 契约、演示脚本和前后端说明与当前实现一致。
- 商业分析：`Aanlyze/` 中四份 Part 2 分析书已按分析师意见完成证据化重写，Markdown 为唯一正文源；共享证据登记区分官方事实、公司披露、行业基准、课堂情景和待验证门槛，四份横向 A4 PDF 输出到 `output/pdf/`。
- 本地启动：根目录 `./start.sh` 按需初始化依赖并统一启动、监控和停止前后端开发服务。
- 平台时间：后端统一使用可注入的 `Asia/Hong_Kong` 时钟，API 返回 `+08:00`，前端全局时钟每秒推进并每 60 秒与服务端重新同步。

- 离线 AI v1：使用 5,376 条合成样本完成 HistGradientBoosting 等待时间回归、时间切分、双基线对比、残差区间和特征重要性；FastAPI 只以影子模式加载，用户结果由课堂 AI v2 或统计降级模型决定。

- 三维与商业展示：首页以四档颜色、收窄后的三层管线、双向错峰粒子密度/速度和口岸两侧累积共同表达路线压力；Tooltip 跟随鼠标/聚焦口岸且不越出模块，聚焦后仍可自由控制，指针、滚轮、触摸或方向键操作会暂停巡航8秒，减少动态效果时关闭连续巡航和镜头飞行。登录、三档套餐、模拟结账、本地收据、商业订阅与 Demo MRR 形成可操作闭环，全部明确为非真实认证、支付和收入。
