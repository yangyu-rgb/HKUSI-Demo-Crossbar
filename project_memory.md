## 当前状态

- 最终产品方向：**CrossBorder AI**，用于深港口岸等待预测和跨境通勤规划的 Demo。
- 本地分支：`main`，跟踪 `origin/main`。
- 2026-07-09 已成功刷新远端引用；模块化工作开始前，本地与远端均位于 `d29c228`。
- 后端：采用 Router、Schema、Service、Repository 分层的 FastAPI 应用，具备统一的请求 ID 错误结构和本地 SQLite 持久化。
- 前端：七个桌面业务路由加独立 `/mobile/*` 手机个人应用；手机端具备自己的布局、五栏导航、会话恢复、路线规划、无副作用场景推演、关联众包反馈、提醒通知和模型简报。
- 路线规划：香港→深圳与深圳→香港均可用；可解释时间加权统计模型按工作日/周末、小时、模拟天气和 28 天历史半衰期生成基线，并以衰减众包修正、动态不确定性计算 90% 置信区间和迟到概率。

- 文档：API 契约、OpenAPI Schema、生成的 TypeScript 契约、演示脚本和前后端说明与当前实现一致。
- 本地启动：根目录 `./start.sh` 按需初始化依赖并统一启动、监控和停止前后端开发服务。
- 平台时间：后端统一使用可注入的 `Asia/Hong_Kong` 时钟，API 返回 `+08:00`，前端全局时钟每秒推进并每 60 秒与服务端重新同步。

- 离线 AI v1：使用 5,376 条合成样本完成 HistGradientBoosting 等待时间回归、时间切分、双基线对比、残差区间和特征重要性；FastAPI 只以影子模式加载，用户结果由课堂 AI v2 或统计降级模型决定。

## 已确定决策

- CrossBorder AI 是可执行代码、样例数据和辅助文档中的唯一产品主题。
- 原型保持确定性并仅用于 Demo，不依赖真实 i口岸、地图、交通、天气、通知或外部 AI 服务。
- Demo 预测器使用 AI v2.2 基础 HGB、透明场景乘数、官方等级、最大30%衰减众包和深圳一致性区间核验；所有中间值进入预测依据。
- 反馈、订阅和企业方案历史保存在本地 SQLite；模拟元数据保存在 JSON；显式重置接口按当时香港时间恢复相对时间种子。
- 预测 API 边界使用稳定地点 ID，`/api/predict` 不接受自由文本地点。
- 根据香港当前时间计算预计到达时间，并在最晚出发建议中加入十分钟安全缓冲。
- TanStack Query 负责共享服务端状态、轮询、Mutation 和缓存失效；临时表单和编辑状态保留在功能页面内。
- 前端契约类型由后端 OpenAPI 文档生成，不重复手写响应接口。
- B2B SaaS 面向跨境巴士、物流和企业 HR，是主要商业方向；个人流程用于展示产品价值和数据闭环。
- `AGENTS.md` 保存仓库操作规则；根目录 `project_memory.md` 保存任务历史、验证、同步说明和后续事项。
- 仓库自有说明文档使用中文；框架名、命令、路径、API 名称和字段等专有名词保留原文。
- 项目记忆仅记录日期、任务、修改范围、验证结果和必要的同步说明。
- 众包反馈质量由新鲜度、等待偏差和人流一致性确定；原始记录保留在 SQLite，过期数据不显示也不参与预测。
- 本地开发默认从仓库根目录运行 `./start.sh`；脚本缺少依赖时自动初始化，独立服务命令仅用于调试。
- Agent 可以执行必要的 Git 只读检查和上游刷新，但不执行 `git add`、`git commit` 或 `git push`；任务结束时只提供建议的 commit 文案。
- 用户结果使用香港官方历史客流压力和新鲜官方拥堵等级；地点、交通、天气和事件 Provider 仍为本地 Demo 输入。
- AI v1 只预测各口岸等待分钟，不负责路线、预算或推荐排序；`crowd_level` 由目标生成，因此禁止作为模型特征。
- AI v1 使用前 39 天训练、随后 8 天验证、最后 9 天测试；合成数据指标不得作为真实效果声明，获得真实数据后必须重新划分并评估。
- AI v1 二进制保存在被忽略的 `data/runtime/models/`，仓库仅保存可复现训练代码、元数据和报告；FastAPI 仅在校验模型、特征和训练数据哈希后进行影子计算，不参与 AI v2 的用户推荐。
- 课堂 AI v2.2 明确标注 `public_data_hybrid_classroom_demo`；香港真实客流作为基础特征，等待分钟仍是生成目标。路线、预算和偏好由确定性优化层处理，产物不可用时回退统计模型。
- AI v2.2 固定按模型基线、场景乘数、官方等级、最多30%众包、深圳核验和不确定性合并执行；降级路径禁止重复校准。
- 场景 A/B 对比使用草稿覆盖和默认场景各运行一次无记录预测，不保存场景、预测运行、影子观测或审计；只有显式保存才改变后续业务流程。
- 影子观测按预测的口岸写入本地 SQLite，记录统计值、AI 值、差异和不可用原因；演示重置会清空观测，写入失败也不影响用户预测。
- 订阅预览选择下一次有效通勤日期，并从到达前三小时内模拟正式预测；它展示三类提醒是否触发，但不发送通知也不持久化历史。
- 企业批量预测使用批次默认偏好/预算，员工可显式覆盖；结果回显实际采用的配置，不再根据员工序号分配偏好。
- 节假日配置用于选择匹配历史样本；重复事件按影响等级调整指定口岸的统计基线并出现在预测依据与实时异常中。AI v1 尚无对应事件特征，仅在影子模式比较。
- 提醒预览始终是纯计算；只有显式保存的评估才进入 SQLite 历史并拥有已读状态，因此不会把用户查看行为伪装成真实通知投递。
- 本地 JSON Provider 统一输出来源、读取时间、版本、状态和是否降级；当前 `available` 仅表示本地样例文件可用，绝不表示真实口岸或天气数据可用。
- 正式预测按查询、时间、模型和输入版本生成稳定 `forecast_run_id`。关联反馈只用于课堂校准，不进入训练标签。
- 现场真实标签、建模同意、V2 生产 readiness 和生产晋级已经退出课程范围；API、页面和启动流程不得重新引入。
- 生成基础等待目标只用于课堂工程评估，指标不得包装为真实口岸效果；AI v1 冻结为影子对照，统计模型负责自动降级。
- 本地 Demo 提供运营、通勤者和企业管理员三种身份；企业方案按组织隔离，通勤者不可访问企业功能，审计仅运营身份可见。
- 本地告警周期和通知收件箱闭合提醒演示，但不宣称已接入邮件、短信或推送。
- 官方拥堵等级和每日客流只能作特征或外部校验，不能用区间中点或网站估计值伪造实际分钟。
- 香港入境处主特征与深圳市口岸办公开核验快照登记为 `approved_feature_only`；i口岸内部接口保持 `blocked`。
- `start.sh` 不启动采集器，只使用仓库快照；维护采集为显式可选工具，断网不得阻塞 Demo。
- 官方当前值与追加式修订历史分离；预测只使用 `generated_at` 当时已获知的数据并冻结到运行，后续修订不得回填历史预测。
- 官方等级可按权重校准课堂分钟估算，但外部评估仍只计算类别命中率、序数误差和混淆矩阵；禁止据此生成伪分钟 MAE 或增加真实标签计数。
- Playwright 必须分别等待后端存活探针和前端页面；GitHub CI 使用单 worker，避免共享 Demo SQLite 的 reset 竞态。
- 开发任务统一在 `README.md` 中按高、中、低优先级使用 Markdown 复选框维护；完成任务时同步勾选并更新本文件。
